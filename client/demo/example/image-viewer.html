Image Viewer

A drawing canvas offers view-box functionalities (pan, zoom, rotate), optionally in combination with a background image.

View-box Functionalities

Programmatically

The view-box of a drawing canvas can be reached with the drawing.getViewbox() function.

There are numerous view-box functions available. For example, drawing.getViewbox().rotateRight(), rotates the view-box by 90Â° clockwise; drawing.getViewbox().fit() makes the whole view-box visible. See the API reference for more details.

Gestures

The view-box listens on browser events. With mouse events, one can pan (click and drag), zoom (mouse-wheel) and rotate (double-click) the image. There is also partial support for mobile touch devices: panning the image with touch and drag works, zooming and rotating is not yet supported.

Pan-zoom-rotate gestures are available for the following modes:

  • drawing.setModeHover()
  • drawing.setModeModify()
  • drawing.setModeSelect()
  • drawing.setModeView()
  • drawing.setModeHover()

And partially for drawing.setModeCreate() (only zooming is possible here).

Background Image

The drawing.setBackgroundImage(url, opt_callback) function sets a background image to a drawing canvas. The function expects an image URL as its first parameter. The second (optional) parameter expects a function, a so called callback function, that is executed after the image is loaded completely.

Combined Usage

A typical problem when implementing an image viewer is to fit an image into the drawing canvas as soon as the background image is loaded completely. This can be done by calling the drawing.getViewbox().fit() function from inside the callback function, e.g., in this way: drawing.setBackgroundImage(url, function() { drawing.getViewbox().fit(); }).
