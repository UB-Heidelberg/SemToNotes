Browser Support

A drawing canvas works with three different browser rendering engines: Canvas, SVG and VML.

If no special rendering engine is needed, the drawing canvas itself searches for the best engine available, Canvas, SVG, or VML, in this order.

In seldom cases it might be useful to force a drawing canvas to use a specific rendering engine. Rendering engines can be forced within the construction of a drawing canvas object, e.g., by calling new xrx.drawing.Drawing(element1, xrx.engine.CANVAS), new xrx.drawing.Drawing(element2, xrx.engine.SVG) or new xrx.drawing.Drawing(element3, xrx.engine.VML).

It is recommended to wrap a constructor call with a xrx.engine.isSupported(xrx.engine.CANVAS), xrx.engine.isSupported(xrx.engine.SVG) or xrx.engine.isSupported(xrx.engine.VML) condition if one forces a special rendering engine.

It is also recommended to include a fall-back message inside the drawing canvas' <div> element, e.g., <div id="drawing1"><span>Your browser does not support HTML Canvas rendering.</span></div>. The fall-back message is automatically removed from the drawing canvas as soon as an engine is initialized successfully.
